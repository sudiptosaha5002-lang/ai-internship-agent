System Architecture

Welcome to the InternshipAgent technical documentation. Here is a breakdown of how the various components of the platform work together to scan, process, and match you with your perfect internship.

Frontend Architecture

The user interface is built securely with modern web standards, focusing on a responsive and animated experience.

Main Dashboard

The core landing page. It houses the primary UI where users can initiate AI scans to find jobs, view their matched internships in real-time, and access the Mock Interview practice panel.

Client-Side Logic

Handles all frontend interactivity. It manages the simulated AI scanning animations, the drag-and-drop resume upload UI, and the dynamic mock interview chat bot interface.

Design System

Contains the global glassmorphic design language, liquid background animations, and responsive layouts used across the platform.

Ancillary Pages

Ancillary pages structured dynamically using CSS grid and flexbox, styled securely to inform users on data privacy patterns and facilitate contact.

Backend Engine

The underlying Python architecture responsible for data aggregation and AI matching.

The Core Brain

The main application entry point. It acts as the bridge that fetches scraped jobs from the cloud database and prepares them for the Google Gemini agent matching module.

Data Aggregation

Uses the Jina AI Reader API to systematically fetch real internship listings from the web, convert them into clean formatted text, and upload them to the MongoDB Cloud DB.

Document Parser

Takes the user's uploaded PDF resume, extracts the raw textual data using PyPDF2, and stores it in the database so the AI understands the user's unique technical profile.

Cloud Infrastructure

Manages the secure connection to the remote MongoDB Atlas cluster, exposing access to the internal `jobs_col` and `resumes_col` collections for the other scripts.
